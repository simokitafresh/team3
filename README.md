# 📱 SmartDrill – AI時代の暗記学習支援アプリ

**教材をスマホで撮るだけで、自動的に学習が始まる。**  
繰り返し学習で「覚えるまで」をサポートする、シンプルで続けやすい学習ツール。

---

## 🎯 プロジェクト概要

SmartDrillは、学習プリントや教科書をスマートフォンで撮影するだけで、自動的に一問一答形式の問題を生成し、効率的な反復学習を可能にするWebアプリケーションです。

**MVP（最小実装版）では**：
- **📸 スマホ撮影→問題化のデモ機能**（プレゼンのキラー機能）
- 英単語テストを撮影して自動で問題生成
- 間違えた問題を優先的に再出題する基本的な個別最適化
- スマホでもPCでも使えるレスポンシブなUI

---

## 🚀 なぜこのプロジェクトを作るのか？

### 理由1：学習現場の普遍的な課題解決
多くの学習現場で共通して見られる以下の課題に着目しました：
- **教材の管理問題**：プリントの紛失、整理の煩雑さ
- **学習の非効率性**：苦手な問題と得意な問題を同じ頻度で復習
- **進捗の不可視化**：どこまで覚えたか、何が苦手かが不明確

### 理由2：実践的な技術チャレンジ
- プログラミングスクールで学んだ技術の総合的な活用
- **10日間という制約**の中で「動くもの」を完成させる実践力
- チーム開発の経験を積む貴重な機会

### 理由3：持続可能な学習支援の仕組み作り
- 「使う人が無理なく続けられる」シンプルな設計
- テクノロジーを活用した学習体験の向上
- 将来的な拡張性を見据えた基盤構築

## 🎭 プレゼン戦略（チーム内共有）

### 🌟 キラー機能：「撮影→問題化」デモ

**なぜこれが差別化要因になるか**
1. **視覚的インパクト**
   - 「入力の手間」という最大の障壁を一瞬で解決
   - 他チームの「フォーム入力」との圧倒的な差
   - 「未来感」のある体験

2. **30秒でつかむ心**
   ```
   0-10秒：「また単語テストなくした...」（共感）
   10-20秒：スマホで撮影→問題生成（驚き）
   20-30秒：「もう なくさない、忘れない」（解決）
   ```

### 📱 デモシナリオ（具体的な流れ）

**事前準備**
- 中学英単語テスト（見やすいフォントで印刷）
- 3パターンの異なるレイアウト
- 各20単語程度（apple, book, school など基礎単語）

**デモの流れ**
1. **導入**（10秒）
   - 「これ、昨日子どもが持って帰ってきた単語テストです」
   - プリントを見せる

2. **撮影**（10秒）
   - スマホを取り出して撮影
   - 「解析中...」のローディング画面（2-3秒）
   - ✨ 問題が生成される瞬間を演出

3. **学習開始**（10秒）
   - 「appleの意味は？」→ りんご → ⭕ 正解！
   - 「次の問題へ」のスムーズな遷移

4. **締め**（10秒）
   - 「間違えた問題は後でまた出題されます」
   - 「これで、もう単語帳をなくす心配はありません」

### 🔧 技術的実現方法

**プレゼン用の確実な動作**
1. **事前登録方式**
   - 特定の画像を事前に登録
   - 画像ハッシュまたは簡易的な特徴量で識別
   - 該当する問題セットを即座に表示

2. **フェイルセーフ**
   - 認識できない場合は「サンプル問題」に切り替え
   - 「よく似た問題を見つけました」でカバー

3. **演出面の工夫**
   - カメラ起動→撮影のスムーズなUI
   - プログレスバーで「解析感」を演出
   - 成功時の効果音/アニメーション

### なぜ「親目線」のストーリーが効果的か

1. **共感を生む具体性**
   - 抽象的な「学習効率化」より「単語帳をなくした」の方が場面を想像しやすい
   - 聴衆（講師・他の受講生）も似た経験を思い出しやすい

2. **課題の深刻さを自然に伝える**
   - 統計データより、実体験の方が印象に残る
   - 「技術で解決したい」という動機が明確

3. **差別化要因**
   - 多くのチームが技術中心で語る中、人間中心のアプローチは記憶に残る
   - 「なぜ作るか」が明確なプロジェクトは評価されやすい

### プレゼンでの使い分け案
- **導入部**：親目線ストーリーで感情に訴える（30秒）
- **デモ**：撮影→問題化の魔法の瞬間（40秒）
- **技術説明**：どう実現したか（90秒）
- **締め**：「こんな未来を作りたい」ビジョン（30秒）

## 🔧 技術スタック（MVP）

### コア技術
- **バックエンド**：Python 3.x + Flask（軽量で学習曲線が緩やか）
- **フロントエンド**：HTML + Bootstrap 5 + Jinja2（レスポンシブ対応）
- **データ管理**：CSV/JSONファイル（初期は静的データで安定性重視）
- **状態管理**：Flaskセッション（ユーザーごとの学習進捗を記録）

### 撮影→問題化デモの実装
- **画像アップロード**：Flask-Upload + Pillow
- **画像識別**：事前登録画像のハッシュ値マッチング
- **デモ用データ**：英単語テスト3パターンを事前準備
- **フォールバック**：未認識時はサンプル問題セットを表示

### 将来的な拡張構想
- OCR連携（Tesseract OCR）
- AI問題生成（OpenAI API）
- データベース化（SQLite → PostgreSQL）
- ユーザー認証システム

## ✅ スコープ定義（MVP段階）

### やること ✅
1. **📸 撮影→問題化デモ機能**
   - スマホカメラでの撮影UI
   - 画像アップロード処理
   - 事前準備した英単語テストの認識
   - 問題自動生成の演出

2. **基本的な一問一答機能**
   - 問題表示 → 解答入力 → 正誤判定のサイクル
   - 正答率に基づく再出題ロジック

3. **使いやすいUI**
   - スマホでも快適に使えるレスポンシブデザイン
   - 直感的な操作フロー
   - 撮影ボタンを目立つ位置に配置

4. **学習進捗の可視化**
   - セッション内での正答率表示
   - 苦手問題のハイライト

5. **デモ用コンテンツ**
   - 英単語テスト画像（3パターン）
   - 各パターンに対応する問題セット（各20問）
   - フォールバック用の汎用問題セット

### やらないこと ❌
- ユーザー登録・ログイン機能
- 完全自動のOCR（事前登録画像のみ対応）
- 手書き文字の認識
- 複数科目の切り替え機能
- AIによる問題難易度調整
- 学習履歴の永続保存
